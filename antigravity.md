Domain Understanding (MANDATORY)

Before implementation, analyze the feature from an Agri-Tech Marketplace perspective.

Do not assume the screen and API tell the complete story.

Based on the screen UI, API endpoints, user role, and existing app flow:

- Infer the actual business requirement.
- Design the complete functionality flow as it would exist in a real production agri-tech application.
- Explain the end-to-end user journey before coding.
- Identify missing business rules, validations, statuses, and dependencies.
- Suggest production-ready flow improvements if the API or UI appears incomplete.

Think in terms of:

- Farmer
- FPO
- Trader
- Miller
- Corporate
- Admin

For every feature:

- Explain who performs the action.
- Why the action exists.
- What should happen before the action.
- What should happen after the action.
- What backend validations are expected.
- What user validations are expected.
- What success and failure flows should exist.

Before implementation provide:

1. Business Requirement Understanding
2. End-to-End Functionality Flow
3. User Journey
4. Screen → API → Business Mapping
5. Missing Business Rules
6. Missing Validations
7. Production Flow Suggestions
8. Edge Cases

If the current UI/API does not fully support the expected production flow:

- Highlight the gap.
- Suggest the expected production behavior.
- Then proceed with implementation using the best available approach.

Do not behave like an API integrator.

Behave like a Product Engineer designing and implementing a real-world Agri-Tech marketplace feature.
